Detailed Description

Implements the UNIX file API ( fd streams )

The streams programming interface provides an unified way of transferring data. It wraps the VFS API and implements pipes. For files it keeps track of the current offset.

Slightly different terminology is used here:

UNIX - POSNK

  • File description - stream info
  • File descriptor - stream pointer
  • File (opened) - stream

off_t _sys_lseek ( int  fd,
off_t  offset,
int  whence 
)

Reposition read/write file offset.

The way the new file offset is calculated is determined by whence:

Whence:

  • SEEK_SET
    The offset is set to offset bytes
  • SEEK_CUR
    The offset is set to its current location plus offset bytes
  • SEEK_END
    The offset is set to the size of the file plus offset bytes

Setting an offset past the end of the file will not grow the file until the next write() to the file

int _sys_open ( char *  path,
int  flags,
mode_t  mode 
)

Open a file stream.

The behaviour of this function is determined by flags
It is required to specify one of the access mode flags:

  • O_RDONLY Open read-only
  • O_WRONLY Open write-only
  • O_RDWR Open read-write

Valid optional flags:

  • O_APPEND Open the file for appending

    Before each sys_write the offset is set to the end of the file

  • O_CLOEXEC Set this stream pointer to close on execve(2)

    When this flag is set this stream pointer will be closed when executing a new program.

  • O_CREAT Create file

    If this flag is set and the file does not exist the file will be created using the mode specified in _mode

  • O_NONBLOCK Use non-blocking IO

    When this flag is set _sys_read and _sys_write will not block if no data is available

  • O_TRUNC Truncate file

    If this file already exists and is a regular file and the open mode allows writing the file will be truncated to length 0

  • O_EXCL Ensure the file is created

    If this flag is set and O_CREAT is set and the file already exists _sys_open
    will fail with EEXIST
